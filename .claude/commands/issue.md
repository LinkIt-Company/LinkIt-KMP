사용자에게 어떤 이슈를 만들고 싶은지 물어본 뒤, 이슈 초안을 작성해줘.

## 진행 순서

1. `.github/ISSUE_TEMPLATE/claude_task.md` 파일을 읽어서 이슈 템플릿 형식을 파악한다. (frontmatter 제외, 본문만 사용)
2. 사용자에게 "어떤 이슈를 만들까요? 간단히 설명해주세요." 라고 질문한다.
3. 사용자의 답변을 바탕으로 템플릿 형식에 맞게 이슈 초안을 작성해서 보여준다.
4. 초안을 보여준 뒤 "이대로 이슈를 생성할까요? 수정할 부분이 있으면 말씀해주세요." 라고 확인을 요청한다.
5. 사용자가 승인하면 `gh issue create` 명령어로 GitHub 이슈를 생성한다. 수정 요청이 오면 반영 후 다시 확인을 요청한다.
6. 이슈 생성 후 CLAUDE.md의 브랜치 네이밍 규칙과 브랜치 계층 구조에 따라 작업 브랜치를 만들지 사용자에게 물어본다. 승인하면 해당 규칙에 맞게 새 브랜치를 생성한다.

## 규칙
- 종류는 사용자의 설명을 바탕으로 적절한 항목에 `[x]` 체크한다.
- 관련 모듈 등 선택형 항목도 내용에 맞게 체크한다.
- 완료 조건은 구체적이고 검증 가능하게 작성한다.
- 이슈 생성 시 `gh issue create --title "제목" --body "본문"` 형식을 사용한다.
